1 · System Architecture

All components and data-flow paths. Everything lives in packages/pergola.yaml and is deployed via git pull. The two covers (cover.dach_links / cover.dach_rechts) always move together. Frost thresholds (2.5 °C entry / 3.0 °C exit) are hardcoded constants, not UI-editable inputs. input_boolean.pergola_heating can be set via HA REST API by an external system (e.g. Loxone) — no direct Loxone trigger in the state manager automation.

2 · State Priority Decision Logic

pergola_state_manager evaluates these rules top-down on every trigger. The first matching rule wins. rain_stopped is protected mid-flight — the state manager skips evaluation while the recovery script is running, with three exceptions: (1) frost preempts: temp falls below threshold → cancel recovery script, set frost; (2) rain preempts: rain indicator becomes active → cancel recovery script, set rain; (3) HA restart: re-trigger the recovery script to resume the interrupted drain sequence. frost and user_override both check hourly_rain on exit and route to rain_stopped if wet. Each state box shows the cover action and the condition that causes exit / re-evaluation.

No movement
Tilt → ~74 (vertical, open)
Formula-driven tilt
Recovery script active

3 · Cover Response Automation

pergola_cover_response triggers on every state change and on a 5-minute timer. The 5-minute timer is the only mechanism that moves covers as the sun slowly tracks across the sky within a steady state. The master switch is checked first on every trigger. All cover movement goes through script.pergola_set_slat_angle — the single authoritative slat→tilt_position converter. The cover response checks the deadband before calling the script; the post-rain recovery script bypasses the deadband.

4 · Tilt Calculation Pipeline

Two-step calculation run by script.pergola_set_slat_angle (cover movement) and the template sensors (display). Step 0 converts sun azimuth + elevation into the Effective Sun Angle ($A_{eff}$) — a unified 0°–180° east-to-west scale: 0° = East Horizon → 90° = Zenith/Noon → 180° = West Horizon. $A_{eff}$ and the slat angle sit on the same number line: in heating mode the slat tracks $A_{eff}$ directly; in cooling mode the slat is offset ±90° from $A_{eff}$.
COOLING_LOWER_BOUND ≈ 15.3° — derived at runtime as MaxOpenWest(A_eff_flip) from slat geometry; the ventilation floor used when the back-face strategy can no longer guarantee full shade (MaxOpenEast > max_tilt − 5°, i.e. A_eff ≈ 58°). Fully parametric: re-derives when geometry inputs change.
DEADBAND_TILT = INT(5 / max_tilt × 100) (default 4 tilt units ≈ 4.9°) — the cover response checks this before every move; the post-rain recovery script bypasses it.

5 · Post-Rain Recovery Script

script.pergola_post_rain_recovery runs whenever automation_state enters rain_stopped (rain cleared, frost exited with residual rain, user_override exited with residual rain, or HA restarted with state persisted as rain_stopped). Slat angles (8°, 15°) are converted via script.pergola_set_slat_angle (hardware correction formula applied there). The state manager skips normal evaluation while rain_stopped is active — only this script exits it normally. Exception: if frost or rain triggers mid-sequence, the state manager cancels this script via script.turn_off and immediately sets the higher-priority state.
