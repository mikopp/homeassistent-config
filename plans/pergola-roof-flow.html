1 · System Architecture

All components and data-flow paths. Everything lives in packages/pergola.yaml and is deployed via git pull. The two covers (cover.dach_links / cover.dach_rechts) always move together.

2 · State Priority Decision Logic

pergola_state_manager evaluates these rules top-down on every trigger. The first matching rule wins. rain_stopped is protected mid-flight — the state manager skips evaluation while the recovery script is running, with three exceptions: (1) frost preempts: temp falls below threshold → cancel recovery script, set frost; (2) rain preempts: rain indicator becomes active → cancel recovery script, set rain; (3) HA restart: re-trigger the recovery script to resume the interrupted drain sequence. frost and user_override both check hourly_rain on exit and route to rain_stopped if wet. Each state box shows the cover action and the condition that causes exit / re-evaluation.

No movement
Tilt → 71 (vertical, open)
Formula-driven tilt
Recovery script active

3 · Cover Response Automation

pergola_cover_response triggers on every state change and on a 5-minute timer. The 5-minute timer is the only mechanism that moves covers as the sun slowly tracks across the sky within a steady state. The master switch is checked first on every trigger.

4 · Tilt Calculation Pipeline

Two-step calculation run by pergola_cover_response and the template sensors. Step 1 computes the desired slat_angle (degrees) from sun geometry and season. Step 2 applies hardware motor correction to produce tilt_position (0–100) sent to the covers.
threshold = max_tilt_angle − 180 = 122 − 180 = −58° (default). perfect_angle sign convention: positive = sun from west, negative = sun from east.

5 · Post-Rain Recovery Script

script.pergola_post_rain_recovery runs whenever automation_state enters rain_stopped (rain cleared, frost exited with residual rain, user_override exited with residual rain, or HA restarted with state persisted as rain_stopped). Slat angles (8°, 15°) are converted via the hardware correction formula before sending to covers. The state manager skips normal evaluation while rain_stopped is active — only this script exits it normally. Exception: if frost or rain triggers mid-sequence, the state manager cancels this script via script.turn_off and immediately sets the higher-priority state.

    ENTER(["automation_state enters: rain_stopped\nFrom: rain clears — both indicators off\nOR: frost exits with hourly_rain > 0\nOR: user_override exits with hourly_rain > 0\nOR: HA restart with state persisted as rain_stopped"])

    W30["Wait 30 s\nSystem stabilisation"]
    T8["set_cover_tilt_position → 8°\nboth covers (tilt_position ≈ 13)\nSlight opening to start water drainage"]
